# 3D Brick Breaker — 프로그램 설계 보고서

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 3D Brick Breaker |
| 개발 언어 | C++ (C++20) |
| 그래픽 라이브러리 | OpenGL / FreeGLUT / GLU |
| 개발 환경 | Visual Studio 2022, Windows 10/11 x64 |
| 빌드 방식 | MSVC (v145), Release x64, Static CRT (/MT) |

### 게임 소개

3×3×3 구조의 3D 벽돌 덩어리 중앙에 위치한 **파란색 코어 블록**을 공으로 파괴하는 것이 목표인 게임이다.  
플레이어는 마우스 드래그로 블록 덩어리를 회전시키고, 키보드로 경계 벽을 조작하여 공이 코어에 맞도록 유도한다.

### 게임 모드

| 모드 | 설명 |
|------|------|
| **스토리 모드** | 3개 스테이지를 순서대로 진행. 총 클리어 시간이 기록으로 저장됨 |
| **무한 모드** | 스테이지가 끝없이 생성. 도달한 최고 웨이브 수가 기록으로 저장됨 |

## 4. 함수별 상세 설명

### 4-1. DLL 내장 로더

#### `ExtractFreeglut()`

| 항목 | 내용 |
|------|------|
| 역할 | exe 리소스에 내장된 `freeglut.dll`을 `%TEMP%` 폴더로 추출 |
| 호출 시점 | `main()` 진입 직후, GLUT 관련 함수 호출 전 |
| 처리 과정 | `FindResourceW` → `LoadResource` → `LockResource` → `CreateFileA` / `WriteFile` |

배포 시 DLL 파일을 별도로 동봉하지 않아도 되도록, 빌드 단계에서 `freeglut.dll`을 RCDATA 리소스로 exe 안에 내장한다. 프로그램 실행 시 이 함수가 이진 데이터를 임시 폴더에 파일로 복원한다.

#### `DliHook()`

| 항목 | 내용 |
|------|------|
| 역할 | Windows Delay-Load 메커니즘이 `freeglut.dll`을 찾을 때 가로채어 추출 경로에서 로드 |
| 등록 방법 | `__pfnDliNotifyHook2` 전역 포인터에 할당 |
| 효과 | exe 실행 파일 하나만으로 배포 가능 |

### 4-2. 파일 입출력

#### `loadHighScore()`

`score.txt`를 읽어 `bestTimeSeconds`(스토리 최단 시간)와 `bestWave`(무한 최고 웨이브)를 로드한다. 파일이 없거나 형식이 맞지 않으면 기본값(9999초, 0웨이브)을 유지한다.

#### `saveHighScore()`

기록 갱신 시점(스토리 전체 클리어 또는 무한 모드 웨이브 갱신)에 현재 `bestTimeSeconds`와 `bestWave`를 `score.txt`에 저장한다.

### 4-3. 초기화

#### `init()`

| 설정 항목 | 내용 |
|-----------|------|
| 배경색 | (0.05, 0.05, 0.10) — 어두운 남색 |
| 깊이 테스트 | `GL_DEPTH_TEST` 활성화 |
| 조명 | `GL_LIGHT0`, 위치 (10, 10, 10), 흰색 Diffuse |
| 컬러 재질 | `GL_COLOR_MATERIAL` 활성화 |
| 별 파티클 | 200개 랜덤 위치 초기화 |
| 기록 로드 | `loadHighScore()` 호출 |

#### `initStage(stage)`

스테이지 시작 시 공 위치·속도, 블록 회전값, 스킬 쿨타임을 초기화하고, 스테이지 번호 또는 모드에 따라 `activeBricks[3][3][3]` 배열을 아래 규칙으로 설정한다.

| 조건 | 활성화 패턴 |
|------|------------|
| 스토리 1스테이지 | 세 평면의 중심 교차선(십자 형태, 13개) |
| 스토리 2스테이지 | 꼭짓점 8개 제외 (19개) |
| 스토리 3스테이지 | 전체 27개 |
| 무한 모드 | 75% 확률 랜덤 (코어 제외) |
| 공통 | 중앙 코어 `[1][1][1]`은 항상 활성 |

### 4-4. 게임 로직

#### `checkCollision()` — 충돌 감지

공의 2D 위치와 각 활성 벽돌의 3D 위치(회전 적용 후 Z=0 투영)의 거리를 계산한다.

**충돌 판정 알고리즘:**

1. 현재 `blockRotX`, `blockRotY`를 라디안 변환
2. 각 벽돌 로컬 좌표에 Y축 회전 → X축 회전 행렬 순서로 적용
3. 공과 회전된 벽돌 중심 간 3D 유클리드 거리 계산  
   (`dist = √(dx² + dy² + dz²)`)
4. `dist < 0.9f`이면 충돌 판정, 벽돌 비활성화 후 속도 반전

**충돌 후 속도 처리:**

| 조건 | 처리 |
|------|------|
| 현재 속도 > 0.2 (고속) | 속도 × (−0.5): 반전 + 감속 |
| 현재 속도 ≤ 0.2 (저속) | 속도 × (−1.02): 반전 + 미세 가속 |

#### `checkStageClear()` — 클리어 판정

코어 `activeBricks[1][1][1]`가 `false`가 된 순간을 감지하여 모드별로 분기한다.

- **스토리 모드**: `currentStage < maxStage`이면 `STAGE_CLEAR`, 아니면 `ALL_CLEAR` 전환 및 기록 갱신
- **무한 모드**: `currentStage++` 후 즉시 `initStage()` 호출로 다음 웨이브 시작

### 4-5. 렌더링

#### `drawAndMoveStars()`

200개의 별 파티클을 GL_POINTS로 렌더링하며 매 프레임 Z 좌표를 +0.05씩 이동시킨다. 별이 카메라 근처(Z > 10)를 지나면 Z = −30으로 재배치하여 무한 흐름 효과를 만든다. Z 거리에 따른 밝기 보정으로 원근감을 표현한다.

#### `drawOctagonWall()`

반지름 7.5의 정팔각형 경계 벽을 렌더링한다.

- `sin`/`cos` 시간 함수로 RGB 색상이 주기적으로 변화
- 두께 1~3의 선 3겹으로 글로우(glow) 효과 구현 (GL_BLEND 사용)
- `wallRotZ` 값만큼 Z축 회전 (A/D 키 조작)

#### `draw3x3x3Bricks()`

`blockRotX`, `blockRotY`로 전체 블록 덩어리를 회전시킨 후 활성화된 벽돌만 렌더링한다.

- 중앙 코어(0,0,0): 파란색 (RGB 0.2, 0.4, 1.0)
- 일반 벽돌: 회색 (RGB 0.3, 0.3, 0.4)
- 모든 벽돌에 `glutWireCube`로 테두리 추가

#### `drawAndMoveBall()`

매 프레임 공의 물리 시뮬레이션을 수행한다.

```
[물리 처리 순서]
1. ballSpeedY += gravity          (중력 적용)
2. ballX += ballSpeedX            (위치 갱신)
   ballY += ballSpeedY
3. 원형 벽(r=7.0) 충돌 검사
   → 법선 반사: v' = v − 2(v·n)n
   → 반사 후 속도 × 1.02 (가속)
4. checkCollision()                (벽돌 충돌)
5. checkStageClear()               (클리어 확인)
6. 최대 속도 초과 시 × 0.95 (제한)
7. draw2DBall(0.4f)                (렌더링)
```

#### `draw2DBall(radius)`

GL_TRIANGLE_FAN을 이용해 30각형으로 원을 근사하여 공을 렌더링한다. 조명 영향을 받지 않도록 `GL_LIGHTING`을 임시 비활성화한다.

#### `renderText(x, y, font, string, r, g, b)`

GLUT 비트맵 폰트로 임의 2D 위치에 문자열을 출력하는 헬퍼 함수이다. `drawUI()` 내에서 반복 사용된다.

#### `drawUI()`

3D 투영을 `gluOrtho2D`로 잠시 2D 좌표계로 전환한 후 상태별 HUD를 그린다.

| 상태 | 출력 내용 |
|------|----------|
| MENU | 타이틀, 스토리 최고 기록, 무한 최고 웨이브, 모드 선택 안내 |
| INSTRUCTION | 마우스/키/Z 스킬 조작법 |
| PLAYING | 스테이지(웨이브) 번호, 경과 시간 (MM:SS.mmm), Z 스킬 쿨타임 |
| STAGE_CLEAR | "STAGE N CLEAR!" 메시지 |
| ALL_CLEAR | 축하 메시지, 최종 클리어 시간 |

### 4-6. 입력 처리

#### `keyboard(key, x, y)`

| 상태 | 키 | 동작 |
|------|----|------|
| MENU | `1` | 스토리 모드 시작 |
| MENU | `2` | 무한 모드 시작 |
| MENU | `3` | 조작법 화면 전환 |
| INSTRUCTION | ESC / ENTER | 메뉴 복귀 |
| STAGE_CLEAR | ENTER | 다음 스테이지 시작 |
| ALL_CLEAR | ENTER | 메뉴 복귀 |
| PLAYING | `A` / `D` | 팔각형 벽 ±5도 회전 |
| PLAYING | ESC | 무한 모드 종료 후 메뉴 복귀 |
| PLAYING | `Z` | 필살기: 공을 목표로 대시 (10초 쿨타임) |

**Z 스킬 로직:**  
1회차 사용 시 공이 중앙 코어(0, 0)를 향해 대시한다. 이후 사용부터는 랜덤 좌표로 방향을 설정하되, 공과의 거리가 0.5 미만이면 목표를 1.5 보정하여 제자리 대시를 방지한다.

#### `mouse(button, state, x, y)`

PLAYING 상태에서 좌클릭 누름/뗌에 따라 `isDragging` 플래그와 드래그 기준 좌표를 갱신한다.

#### `motion(x, y)`

`isDragging`이 활성화된 동안 마우스 이동량(픽셀)을 0.5배 스케일링하여 `blockRotX`(상하), `blockRotY`(좌우)에 반영한다.

### 4-7. GLUT 콜백

#### `display()`

매 프레임 호출되는 메인 렌더링 함수이다.

```
glClear() → gluLookAt() → drawAndMoveStars()
         → (PLAYING/STAGE_CLEAR) drawOctagonWall() / draw3x3x3Bricks() / drawAndMoveBall()
         → drawUI()
         → glutSwapBuffers()
```

카메라는 (0, 5, 18) 위치에서 원점을 바라보는 고정 시점을 사용한다.

#### `timer(value)`

`glutTimerFunc(16, timer, 0)`으로 자신을 재귀 등록하여 약 62.5fps의 고정 프레임 루프를 구성한다.

#### `reshape(w, h)`

윈도우 크기 변경 시 뷰포트와 투영 행렬을 갱신한다. 투영 설정: FOV 45°, 종횡비 w/h, near=1.0, far=100.0.

## 6. 빌드 및 배포

### 빌드 설정

| 항목 | 값 |
|------|-----|
| 구성 | Release \| x64 |
| 런타임 라이브러리 | `/MT` (Static CRT) |
| 링커 옵션 | `/DELAYLOAD:freeglut.dll` |
| 추가 의존성 | `delayimp.lib` |

### 단일 exe 배포 방식

`resources.rc`를 통해 `freeglut.dll`을 RCDATA 리소스로 exe에 내장한다.  
실행 시 `ExtractFreeglut()`이 `%TEMP%\freeglut.dll`로 자동 추출하고,  
`DliHook()`이 Delay-Load 메커니즘을 가로채 해당 경로에서 DLL을 로드한다.  
따라서 **`Bruck_Breaker.exe` 하나만 배포하면 된다.**

## 7. 결론

본 프로젝트는 OpenGL의 3D 변환(회전 행렬)과 2D UI 렌더링을 혼합하고, GLUT 이벤트 기반 구조 위에 물리 기반 공 시뮬레이션(중력, 반사)과 상태 머신(5가지 GameState)을 구현한 게임이다.  
각 함수는 렌더링, 물리, UI, 입력의 역할을 명확히 분리하여 유지보수성을 확보하였으며, DLL 내장 기법을 통해 단일 실행 파일로 배포 가능한 형태로 완성하였다.
